## [SUMMARY]

[Primary_Requirement]: Create a Python CLI tool that scrapes jobs from LinkedIn/Indeed/Google via JobSpy, analyzes fit against CV using `droid exec`, and provides an interactive review interface for job application tracking.

[Technical_Approach]: Python CLI with `uv` project management, `mise` environment, `typer` for CLI, `sqlite3` for persistence, `JobSpy` for scraping, and `droid exec` for AI analysis. Implements an "intersection strategy" to handle mutually exclusive site filters.

## [IMPLEMENTATION_STRATEGY]
[Execution_Order]: Phase_1 → Phase_2 → Phase_3 → Phase_4 → Phase_5 → Phase_6 → Phase_7

[Critical_Dependency_Chains]:
- Phase_2 (Database) blocks Phase_3 (Scraper) which stores to DB
- Phase_2 (Database) blocks Phase_5 (Analyzer) which stores results
- Phase_4 (CV Parser) should complete before Phase_5 (Analyzer) since analyzer needs CV context
- Phase_6 (CLI) depends on all prior phases

[Risk_Hotspots]:
- [RISK_001]: LinkedIn rate limiting — Mitigation: Use `--auto low` for droid, add delays between scrapes
- [RISK_002]: Filter mutual exclusivity — Mitigation: Intersection strategy implemented in scraper.py
- [RISK_003]: `droid exec` token usage — Mitigation: Batch sizes of 5-10 jobs

[Merge_Conflict_Boundaries]:
- No merge conflicts expected (single developer)
